## Installation

```bash
npx changeelog/reactuse@latest add useArrayIncludes
```

## Usage

```ts
const fruits = ['apple', 'banana', 'cherry'];
const includesApple = useArrayIncludes(fruits, 'apple');
```

## Reference

`useArrayIncludes` is a hook that checks if an array includes a certain value, with options for custom comparison.

### Parameters

The hook accepts three parameters:

<TypeTable
  type={{
    list: {
      description: 'The array to search.',
      type: 'readonly T[] | null | undefined',
    },
    value: {
      description: 'The value to search for.',
      type: 'V',
    },
    options: {
      description: 'Optional configuration object.',
      type: 'UseArrayIncludesOptions<T, V>',
      default: '{}',
    },
  }}
/>

### Options

The `options` object can include the following properties:

<TypeTable
  type={{
    fromIndex: {
      description: 'The position in the array at which to begin searching.',
      type: 'number',
      default: '0',
    },
    comparator: {
      description:
        'Custom comparison function or key of T to use for comparison.',
      type: 'UseArrayIncludesComparatorFn<T, V> | keyof T',
      default: 'undefined',
    },
  }}
/>

### Return Value

The hook returns a boolean indicating whether the value is found in the array.

<TypeTable
  type={{
    returnValue: {
      description: 'True if the value is found, false otherwise.',
      type: 'boolean',
    },
  }}
/>

### Notes

- If list is not an array, a warning is logged and false is returned.
- If a custom comparator function is provided, it's used for comparison instead of the default equality check.
- If comparator is a string, it's used as a key to compare object properties.
- The fromIndex option determines the starting position for the search.
