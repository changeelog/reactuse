## Installation

```bash
npx changeelog/reactuse@latest add useInterval
```

## Usage

```tsx
const { counter, isRunning, pause, resume } = useInterval(1000, {
  callback: (count) => console.log(`Count: ${count}`),
});
```

## Reference

`useInterval` is a hook that creates an interval timer and returns controls for managing it.

### Parameters

The hook accepts two parameters:

<TypeTable
  type={{
    interval: {
      description: 'The interval duration in milliseconds.',
      type: 'number',
    },
    options: {
      description: 'Optional configuration object.',
      type: 'UseIntervalOptions',
      default: '{}',
    },
  }}
/>

### Options

The `options` object can include the following properties:

<TypeTable
  type={{
    immediate: {
      description: 'Determines whether the interval starts immediately.',
      type: 'boolean',
      default: 'true',
    },
    callback: {
      description:
        'Optional callback function to be invoked on each interval tick. It receives the current count as an argument.',
      type: '(count: number) => void',
      default: 'undefined',
    },
    maxCount: {
      description: 'Maximum number of times the interval should run.',
      type: 'number',
      default: 'undefined',
    },
  }}
/>

### Return Value

The hook returns an object with the following properties:

<TypeTable
  type={{
    counter: {
      description: 'The current count of interval ticks.',
      type: 'number',
    },
    isRunning: {
      description: 'Indicates whether the interval is currently running.',
      type: 'boolean',
    },
    reset: {
      description: 'Function to reset the counter to 0.',
      type: '() => void',
    },
    pause: {
      description: 'Function to pause the interval.',
      type: '() => void',
    },
    resume: {
      description: 'Function to resume the interval.',
      type: '() => void',
    },
    setCounter: {
      description: 'Function to manually set the counter value.',
      type: '(value: number) => void',
    },
  }}
/>

## Notes

- The `interval` parameter must be a positive number. If it's not, the hook will throw an error.
- The `setCounter` function only accepts non-negative numbers. Attempting to set a negative value will result in an error.
- If a `maxCount` is specified, the interval will automatically stop running when the counter reaches this value.
- If an error occurs in the callback function, it will be caught and logged to the console, allowing the interval to continue running.
