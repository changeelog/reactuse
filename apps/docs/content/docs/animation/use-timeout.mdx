## Installation

```bash
npx changeelog/reactuse@latest add useTimeout
```

## Usage

```tsx
const { ready, start, stop } = useTimeout(2000, {
  controls: true,
  callback: () => {
    console.log('Timeout triggered');
  },
});
```

## Reference

### useTimeout

`useTimeout` is a hook that creates a timeout and returns controls for managing it.

<TypeTable
  type={{
    interval: {
      description:
        'The timeout interval in milliseconds or a function that returns the interval.',
      type: 'number | (() => number)',
    },
    options: {
      description: 'Configuration object for the timeout behavior.',
      type: 'UseTimeoutOptions',
      default: '{}',
    },
  }}
/>

If the `interval` is not a positive number, an error will be thrown.

### UseTimeoutOptions

An interface defining optional configuration for the timeout:

<TypeTable
  type={{
    controls: {
      description:
        'Determines whether the timeout starts automatically or is controlled manually.',
      type: 'boolean',
      default: 'false',
    },
    callback: {
      description:
        'Optional callback function to be invoked when the timeout triggers.',
      type: '() => void',
      default: 'undefined',
    },
  }}
/>

### Return Value

Returns a `UseTimeoutReturn` object with the following properties:

<TypeTable
  type={{
    ready: {
      description: 'Indicates whether the timeout has completed.',
      type: 'boolean',
    },
    start: {
      description: 'Function to start or restart the timeout.',
      type: '() => void',
    },
    stop: {
      description: 'Function to stop the timeout.',
      type: '() => void',
    },
  }}
/>
