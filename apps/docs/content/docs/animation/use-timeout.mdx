## Installation

```bash
npx changeelog/reactuse@latest add useTimeout
```

## Usage

```tsx
const { isReady, isRunning, start, stop, reset } = useTimeout(2000, {
  callback: () => console.log('Timeout completed'),
});
```

## Reference

`useTimeout` is a hook that provides a controllable timeout mechanism, allowing you to execute a function after a specified delay.

### Parameters

The hook accepts two parameters:

<TypeTable
  type={{
    interval: {
      description:
        'The timeout duration in milliseconds or a function that returns the duration.',
      type: 'number | (() => number)',
    },
    options: {
      description: 'Optional configuration object.',
      type: 'UseTimeoutOptions',
      default: '{}',
    },
  }}
/>

### Options

The `options` object can include the following properties:

<TypeTable
  type={{
    controls: {
      description:
        'Determines whether to enable manual control of the timeout.',
      type: 'boolean',
      default: 'false',
    },
    callback: {
      description: 'The function to be executed when the timeout completes.',
      type: '() => void',
      default: 'undefined',
    },
    autoStart: {
      description: 'Determines whether the timeout should start automatically.',
      type: 'boolean',
      default: 'true',
    },
  }}
/>

### Return Value

The hook returns an object with the following properties:

<TypeTable
  type={{
    isReady: {
      description: 'Indicates whether the timeout has completed.',
      type: 'boolean',
    },
    isRunning: {
      description: 'Indicates whether the timeout is currently running.',
      type: 'boolean',
    },
    start: {
      description: 'Function to start or restart the timeout.',
      type: '() => void',
    },
    stop: {
      description: 'Function to stop the timeout.',
      type: '() => void',
    },
    reset: {
      description: 'Function to reset and restart the timeout.',
      type: '() => void',
    },
  }}
/>

## Notes
- The `interval` parameter can be either a number (in milliseconds) or a function that returns a number.
- If the `interval` resolves to an invalid value (not a number, NaN, or negative), an error will be thrown.
- The `callback` function, if provided, will be executed when the timeout completes.
- When `controls` is set to `true`, you have manual control over the timeout using the start, stop, and reset functions.
- If `autoStart` is `true` and `controls` is `false`, the timeout will start automatically when the component mounts.
- The `reset` function stops the current timeout and starts a new one.
